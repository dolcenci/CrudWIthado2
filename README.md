Summary of Improvements Based on Lecturer Feedback
After reviewing the lecturer’s feedback, several enhancements were made across the system to address usability, organisation, and functionality shortcomings:
1. Lecturers’ Claim Submission – Improved Feature Implementation
Refined the claim submission workflow to remove usability issues.


Ensured all required fields are validated properly before submission.


Improved the layout and structure of the submission screen to make it more intuitive for lecturers.


Enhanced navigation and reduced steps, making the feature smoother and more reliable.


2. Programme Coordinators & Managers’ View – Improved Interface Design
Reorganised the view to provide a clearer, more user-friendly layout.


Grouped related information together to reduce clutter and improve readability.


Added better labels, spacing, and visual hierarchy so users can find information more easily.


Improved the consistency of design elements across the view.


3. Lecturers’ Document Upload – Improved Implementation & Error Handling
Added proper error handling for cases such as unsupported file types, failed uploads, or missing files.


Implemented clear user feedback messages (e.g., success notifications, descriptive error messages).


Enhanced the upload interface to make it simpler and more reliable.


Ensured uploaded documents display correctly and link properly to the related claim.


4. Claim Status Tracking – Strengthened Tracking Functionality
Improved the status update logic to ensure claim statuses refresh accurately and on time.


Enhanced the display of claim progress so users can clearly see the current stage.


Ensured that coordinators, managers, and lecturers all see consistent status information.
